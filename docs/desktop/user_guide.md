# Руководство пользователя

## Начало работы

### Запуск приложения

```bash
cd desktop
python main.py
```

Откроется главное окно приложения с четырьмя вкладками:
- Датасет
- Обучение
- Распознование
- Логи

## Работа с датасетами

### Требования к датасету

Датасет должен содержать:
1. Файл `annotations.json` - аннотации в формате COCO
2. Папку `images/` - исходные изображения

Структура:
```
my_dataset/
├── annotations.json
└── images/
    ├── img_001.jpg
    ├── img_002.jpg
    └── ...
```

### Формат annotations.json

```json
{
    "images": [
        {
            "id": 1,
            "file_name": "img_001.jpg",
            "width": 800,
            "height": 600
        }
    ],
    "annotations": [
        {
            "id": 1,
            "image_id": 1,
            "category_id": 1,
            "segmentation": [[x1,y1,x2,y2,...]],
            "area": 1234,
            "bbox": [x,y,w,h]
        }
    ],
    "categories": [
        {
            "id": 1,
            "name": "tomato_leaf"
        }
    ]
}
```

### Загрузка датасета

1. Перейдите на вкладку **Датасет**
2. Нажмите кнопку **Выберите папку набора данных**
3. Выберите папку, содержащую `annotations.json` и `images/`
4. Приложение отобразит статистику:
   - Количество изображений
   - Количество аннотаций
   - Список классов

## Обучение модели

### Подготовка

Перед обучением убедитесь, что:
- Датасет загружен (см. предыдущий раздел)
- Достаточно свободного места на диске

### Настройка параметров

На вкладке **Обучение** доступны следующие параметры:

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| Эпохи | 10 | Количество проходов по датасету |
| Размер батча | 4 | Количество изображений за один шаг |
| Скорость обучения | 0.001 | Learning rate для Adam |
| Размер изображения | 512 | Размер входного изображения (px) |
| Устройство | cuda/cpu | CPU или GPU для обучения |

### Процесс обучения

1. Настройте параметры
2. Нажмите **Начать обучение**
3. Наблюдайте за прогрессом в:
   - Прогресс-баре
   - Вкладке **Логи**
4. Для остановки нажмите **Остановить тренировку**

### Сохранение модели

После завершения обучения:
1. Нажмите **Сохранить модель**
2. Выберите папку для сохранения
3. Будет создано два файла:
   - `model.pth` - веса модели
   - `model_config.json` - конфигурация

## Распознавание изображений

### Загрузка модели

1. Перейдите на вкладку **Распознование**
2. Нажмите **Загрузить модель**
3. Выберите файл `model.pth`
4. Выберите файл `model_config.json`

### Запуск распознавания

1. Нажмите **Выберите изображение**
2. Выберите изображение (JPG, PNG)
3. Просмотрите результаты:
   - **Исходное изображение** - оригинал
   - **Маска сегментации** - результат сегментации
   - **Наложение** - совмещение оригинала и маски
4. В разделе результатов отображаются обнаруженные классы с уверенностью

## Устранение проблем

### CUDA недоступна

Если при обучении появляется предупреждение "CUDA недоступна":
- Проверьте установку драйверов NVIDIA
- Убедитесь, что установлена версия PyTorch с поддержкой CUDA
- Используйте CPU как устройство

### Ошибки загрузки датасета

- Проверьте наличие `annotations.json` в выбранной папке
- Убедитесь, что пути к изображениям в JSON корректны
- Проверьте наличие всех изображений в папке `images/`

### Недостаточно памяти

- Уменьшите размер батча
- Уменьшите размер изображения
- Используйте GPU с большим объемом памяти

## Рекомендации

### Для лучших результатов

1. **Качество данных**
   - Используйте четкие изображения
   - Обеспечьте разнообразие данных
   - Правильно аннотируйте все объекты

2. **Параметры обучения**
   - Начните с небольшого числа эпох
   - Увеличивайте по необходимости
   - Следите за логами для контроля переобучения

3. **Датасет**
   - Минимум 100 изображений на класс
   - Сбалансированное распределение классов
